### 🛠️ 第一步：开发与运行环境准备

在启动前，请确保您的电脑上已经安装了以下三款必备软件环境：

1. **Docker Desktop**（用于一键跑起 Nacos、MySQL、Redis、RabbitMQ 等中间件）
2. **JDK 21**（后端微服务编译与运行环境）
3. **Node.js (推荐 v20.x 或以上)**（前端 Vue3 项目的运行环境）
4. **Maven 3.9.x**（后端项目依赖打包与管理工具）

------

### 🗄️ 第二步：启动底层基础设施 (中间件容器)

微服务依赖服务发现、缓存、队列和数据库。打开终端，进入项目中的 **`docker`** 目录，执行以下命令：

```bash
# 1. 进入 docker 目录
cd docker
# 2. 一键在后台启动所有中间件容器 (Nacos、MySQL 8.0、Redis 7.0、RabbitMQ)
docker-compose up -d
```

- **验证启动**：在终端输入 `docker ps`，应能看到 `studyroom-nacos`、`studyroom-mysql`、`studyroom-redis`、`studyroom-rabbitmq` 四个容器都在运行（Up 状态）。
- **Nacos 控制台**：浏览器访问 🔗 [http://localhost:8848/nacos](http://localhost:8848/nacos)（账号密码默认为空或 nacos/nacos）。

------

### ☕ 第三步：配置与启动后端微服务集群 (Backend)

后端已被重构为基于 Spring Cloud Alibaba 的分布式多模块微服务架构。

#### 1. 配置 DeepSeek API KEY (AI功能必需)
- **临时指定**：在启动后端服务前，在您的系统环境变量中配置 `DEEPSEEK_API_KEY=您的秘钥`。
- **本地配置文件**：或者在 `code/backend` 各模块下的 `application.yml` 里填入（推荐在 `application-local.yml` 中安全配置并加入 `.gitignore` 防止泄漏）。

#### 2. 一键编译与顺序启动
打开终端，进入 `code/backend` 目录，执行以下步骤：

```bash
# 1. 进入后端代码目录
cd code/backend
# 2. 运行一键编译与分布式启动脚本
start.cmd
```

- **一键脚本说明**：`start.cmd` 会自动执行 `mvn clean package -DskipTests` 对多模块进行打包构建，并在打包成功后，自动为 6 个微服务开启独立的 cmd 窗口，采用错开时间（Stagger）策略依次拉起：
  1. `study-room-user` (:8081)
  2. `study-room-room` (:8082)
  3. `study-room-reservation` (:8083)
  4. `study-room-violation` (:8084)
  5. `study-room-ai` (:8085)
  6. `study-room-gateway` (:8080，网关统一入口)
- **验证启动**：刷新 Nacos 控制台（服务列表），确认 6 个微服务已全部注册成功。

------

### 💻 第四步：启动前端项目 (Frontend)

重新打开一个终端窗口，进入 `code/frontend` 目录，执行以下步骤：

```bash
# 1. 进入前端代码目录
cd code/frontend
# 2. 安装前端依赖包
npm install
# 3. 启动 Vite 本地开发服务器
npm run dev
```

- **验证启动**：终端会输出 `Local: http://localhost:5173/`。
